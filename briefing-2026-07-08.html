1. AI Market Trends

After a run on connector governance, the fresh movement swings back to where the work actually happens — the delivery surface — and this is the biggest surface expansion Anthropic has shipped in weeks.

First, Claude Cowork lands on web and mobile. Cowork is no longer desktop-only: it now runs on web and mobile, with sessions executing remotely in beta so that sessions and files save to the Claude account and follow the user across devices; work continues when the laptop closes, and scheduled tasks run with no device online (Source: Claude by Anthropic, "Claude Cowork on web and mobile"; Claude Help Center release notes — primary). The rollout is described as beginning with the Max plan and widening to more plans over the coming weeks — timing is vendor-stated. This is a genuine change to how an agent engagement is delivered: the agent stops being tied to one machine, which is exactly what makes scheduled and long-running automations dependable.

Second, Microsoft 365 write tools arrive. The Microsoft 365 connector, previously read-and-search, gains write capability: with a Microsoft Entra administrator's consent to the updated permission set and an admin enabling it for the organization, Claude can draft and send email, manage calendar events, update mailbox settings, and create and update files in OneDrive and SharePoint; Teams remains read-only (Source: Claude Help Center release notes; Claude for Microsoft 365 — primary). This crosses the read/write line inside the most common enterprise productivity suite, which is precisely where governance discipline matters most — the admin-consent gate is the control point.

Third, Claude Code and Cowork enter government beta on FedRAMP High. Anthropic put Claude Code and Claude Cowork into public beta inside Claude for Government Desktop, built on the same application commercial customers use but delivered through a FedRAMP High authorized environment: conversation history is stored locally on the agency-managed device, inference runs inside the authorized boundary, administrators can set configuration defaults and control spend across departments, and security teams get tamper-evident audit logs and documentation to support the ATO process, deploying through standard agency MDM (Source: Claude by Anthropic, "Bringing Claude Code and Claude Cowork to government"; Public Sector FAQs — primary). It signals the same desktop file-based agent that commercial teams use is now cleared for the most controlled US public-sector environments.

Fourth, the model calendar settled one item and lands another. Fable 5's usage cap stepped down yesterday, July 7: continued access moved from the included-usage window to usage credits at $10/M input and $50/M output — double Opus 4.8's rate — framed as temporary and capacity-driven (Source: Anthropic, "Redeploying Claude Fable 5"; TechTimes, July 6). Today, July 8, Claude ID verification begins: a flagged subset of consumer users (Free, Pro, Max only; Team, Enterprise, and API exempt) may be asked for a government ID plus a live selfie and facial-geometry template, processed by third-party KYC provider Persona Identities; Anthropic describes the checks as limited to accounts flagged for possible abuse (Source: TechTimes, June 21; it-connect; cybernews). The deployable lineup is otherwise steady — Opus 4.8 proven, Sonnet 5 the Claude Code default and general workhorse at $2/$10 intro — and the frontier did not ship: GPT-5.6 stays gated to roughly 20 government-approved partners with GA "coming weeks," and Gemini 3.5 Pro remains in preview (a July 17 launch is circulating but is a leak, not confirmed by Google) (Source: OpenAI Help Center; TechTimes, July 6; BigGo; Bind AI — vendor/leak-attributed).
